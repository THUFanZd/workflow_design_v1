# 任务描述
我想要实现一个SAE特征解释的workflow，能够给出一个SAE特征的激活(即 输入端)和干预解释(即 输出端)。我现在想完成这个workflow的第6步。
# workflow
1. 从neuronpedia上获取初始观察
2. 根据初始观察，生成解释假说
3. 针对假说，设计测例
4. 进行实验，得到实验结果
5. 将实验结果存储到记忆当中
6. 根据记忆，以及当前轮次的实验结果，对假说进行修正、更改等
7. 重复步骤3-6，直到解释假说收敛，或者达到最大轮次
# 细节描述
## 前置环节
前置环节的代码已经完成。
## 优化假说
这一环节，你需要对输入端和输出端的每个假说，根据实验结果、记忆等参考信息，输出改进后的假说内容，以及改进的理由
1. 输入、工作方式、输出
输入：
  1. 记忆：过去n轮的，记忆文件中记录的内容。当前轮次的实验结果。n是一个默认为1的参数。
提供两种方式，一种输入的记忆为，当前假说在过去n轮的迭代过程。另一种是，当前侧（即 输入侧 或 输出侧）所有假说，在过去n轮的迭代过程。默认第1种。迭代过程从记忆模块的返回结果中提取。其返回结果格式可以参考layer0-feature4-memory.json\logs\0_4\20260314_225001\layer0-feature4-memory.json。你需要实现一个函数，从这个dict中抽取反思环节需要的信息，具体要满足：
（1）保留目前的假说内容和原因
（2）每个假说测例，成功例子的信息只需保留一个，key为成功例子示例；失败例子全部保留。保留分数/正确率等信息
  2. 假说：对于某一侧的假说，根据实验结果，保留得分最高的m个假说（可以作为参数调整，默认为3），作为优化的对象（实际上这一部分在记忆中已经被包括了）
2. prompt设计原则
prompt放到prompts/refine_prompt.py
prompt用英文写
prompt的设计：不能太简陋，要讲清任务背景，输入端的测例要激活SAE特征。
## 工作方式：
调用LLM API，根据上一轮的假说的内容和原因，以及实验的分数/正确率和保留的例子，对输入的假说进行优化。
具体方式为，告诉LLM API简要的背景，给他的假说和原因是之前生成的，实验分数是怎么算的，他要先看分数，再从reason和失败例子中，分析假说的问题是什么，最后给出对已有假说
输出：
和第2步的输出格式一样（第2步目前不要求输出原因，但是memory中提供reason字段了，这一步优化假说的时候，让LLM API按照“原因，假说”的格式给出），输出优化后的假说，使之能进入后续迭代过程。
